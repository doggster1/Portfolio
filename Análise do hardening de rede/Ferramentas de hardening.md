| Tarefa de fortalecimento de segurança (*Hardening*) | Descrição | Usos comuns |
| :--- | :--- | :--- |
| Configurações de linha de base (*Baseline*) | Um conjunto documentado de especificações dentro de um sistema usado como base para futuras compilações, lançamentos e atualizações. | Para restaurar um sistema a uma linha de base anterior após uma interrupção de rede ou alterações não autorizadas na linha de base. |
| Verificações de configuração | Atualização dos padrões de criptografia para dados armazenados em bancos de dados. | Para verificar se há alguma alteração não autorizada no sistema. |
| Desativação de portas não utilizadas | Portas podem ser bloqueadas em firewalls, roteadores, servidores e muito mais para evitar a passagem de tráfego de rede potencialmente perigoso. | Antes da ocorrência de um incidente, para impedir que agentes maliciosos entrem na rede pela porta aberta. Pode ser usado após um incidente para evitar ataques futuros através de portas abertas não utilizadas. |
| Criptografia com os padrões mais recentes | Regras ou métodos usados para ocultar dados de saída e descobrir ou descriptografar os dados de entrada. | Pode ser implementada regularmente para avaliar se os padrões atuais de criptografia são seguros e eficazes para sua organização. Os padrões também podem ser atualizados após uma violação de dados. |
| Manutenção de firewall | A manutenção do firewall envolve verificar e atualizar as configurações de segurança regularmente para se manter à frente de possíveis ameaças. | Isso pode ocorrer regularmente. As regras de firewall podem ser atualizadas em resposta a um evento que permita tráfego anormal na rede. Essa medida ajuda a proteger contra vários ataques DDoS. |
| Descarte de hardware e software | Garante que todo hardware antigo seja devidamente limpo de todos os dados e descartado. | Protege a rede de várias ameaças ao remover software ou hardware desatualizado/não utilizado que não possui as atualizações ou patches de segurança mais recentes. Dispositivos não corrigidos permitem fácil acesso à rede por agentes maliciosos. |
| Autenticação multifatorial (MFA) | Medida de segurança que exige que o usuário verifique sua identidade de duas ou mais maneiras para acessar um sistema ou rede. As opções incluem senha, PIN, crachá, senha de uso único (OTP) enviada ao celular, impressão digital, etc. | Ajuda a proteger contra ataques de força bruta e eventos de segurança semelhantes. A MFA pode ser implementada a qualquer momento e geralmente é configurada uma vez e depois mantida. |
| Privilégios de acesso à rede | Envolve permitir, limitar e/ou bloquear o acesso a ativos de rede para pessoas, funções, grupos, endereços IP, endereços MAC, etc. | Reduz o risco de usuários não autorizados e tráfego externo acessarem a rede interna. Pode ser implementado uma vez ou revisado dependendo do risco de engenharia social ou ataques de força bruta. |
| Análise de logs de rede | Processo de examinar registros (*logs*) de rede para identificar eventos de interesse. | Pode ser configurado para alertar a equipe de segurança sobre tráfego anormal. Pode ser usado preventivamente, durante um evento para rastrear tráfego, ou na resposta a um ataque cibernético. O SIEM é uma ferramenta comum para essa análise. |
| Políticas de senha | As recomendações mais recentes do NIST (Instituto Nacional de Padrões e Tecnologia dos EUA) focam no uso de métodos para adicionar *salt* e *hash* às senhas, em vez de exigir senhas excessivamente complexas ou trocas frequentes. | Usadas para impedir que invasores adivinhem facilmente as senhas dos usuários, seja manualmente ou via scripts que testam milhares de senhas roubadas (ataque de força bruta). |
| Atualizações de *patches* | Atualização de software e sistema operacional (SO) que corrige vulnerabilidades de segurança em um programa ou produto. | Geralmente contêm correções para problemas de segurança. É vital manter os sistemas atualizados porque os invasores são alertados sobre a vulnerabilidade assim que os *patches* são lançados, tentando explorá-la antes que as correções sejam aplicadas. |
| Teste de penetração (*pen test*) | Um ataque simulado que ajuda a identificar vulnerabilidades em sistemas, redes, sites, aplicativos e processos. | Usados para proteger e prevenir contra ataques em potencial. |
| Filtragem de portas | Função de firewall que bloqueia ou permite determinados números de portas para limitar comunicações indesejadas. | Usada para controlar o tráfego de rede e impedir a entrada de potenciais invasores em uma rede privada. |
| Remoção ou desativação de aplicativos e serviços não utilizados | Aplicativos e serviços não utilizados podem se tornar pontos de vulnerabilidade, pois têm menos probabilidade de receber manutenção ou atualizações de segurança. | Procedimento usado para reduzir potenciais vulnerabilidades dentro de uma rede. |
| Backups de servidores e armazenamento de dados | Ajudam a proteger os ativos de dados contra perdas. Os backups podem ser gravados em locais físicos ou enviados/sincronizados para repositórios em nuvem. | Usados para restaurar dados perdidos devido a ataques, erros humanos, falhas de equipamento e outras perdas não planejadas. |
